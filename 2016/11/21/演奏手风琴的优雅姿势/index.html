通过以上的工作，我们 Get 到了重要的数据：子元素的最大高度。接着在列表元素上代理子项的点击事件实现展开收缩效果就 ok 了。说到实现动效的方式，最直接的方式的就是用间歇函数了。作为有志青年，我们当然不能走寻常路，于是 window.requestAnimationFrame() 就进入了我们的视野。

The window.requestAnimationFrame() method tells the browser that you wish to perform an animation and requests that the browser call a specified function to update an animation before the next repaint. The method takes as an argument a callback to be invoked before the repaint.

window.requestAnimationFrame()这个方法是用来在页面重绘之前，通知浏览器调用一个指定的函数，以满足开发者操作动画的需求。这个方法接受一个函数为参，该函数会在重绘前调用。

window.requestAnimationFrame() 优于间歇函数实现的动画在于：

  1. requestAnimationFrame 会把每一帧中的所有DOM操作集中起来，在一次重绘或回流中就完成，并且重绘或回流的时间间隔紧紧跟随浏览器的刷新频率，一般来说，这个频率为每秒60帧。
  2. 在隐藏或不可见的元素中，requestAnimationFrame将不会进行重绘或回流，这当然就意味着更少的的cpu，gpu和内存使用量。

关于该方法的介绍可参见张大的另一篇文章： CSS3动画那么强，requestAnimationFrame还有毛线用？

“命令式动画vs说明式动画”

首先把我们目标拆解下，想要不借助脚本实现 Accordion 的展开/收缩动画效果，就要利用 CSS3 的过渡和转换属性。触发 CSS3 的这些属性一般是通过变化元素的状态来实现，而改变元素状态一般是通过给元素挂载类实现的。但是呢，我们的目标是不借助脚本，所以这里我们会用到一个小技巧：
利用单选按钮（input[type=radio]）和 Label 标签的联动性。

总结下就是：

  1. 利用 CSS3 实现动画效果
  2. 利用单选按钮与 Label 标签的联动性改变元素状态

一、改变元素状态

我们先来看下如何改变元素状态，因为这点在后面会用到。首先 label 是一个很有意思的标签， label 元素不会向用户呈现任何特殊效果，但是它为鼠标用户改进了可用性：当用户选择该标签时，浏览器就会自动将焦点转到和标签相关的表单控件上。也就是说，但我们点击用 label 标签包裹的标题时，就会选中它所对应的目标元素，这样就达到了不用脚本就可以给目标元素挂载类的效果。选择单选按钮作为和标签相关的表单控件是因为，我们希望展开某一项时，其他项能够收缩起来，当然你也可以选择多选按钮以实现多项展开。

二、实现动画效果

这里的方式可就多了，主要看脑洞了。为了方便讲解先做一些约定：待展开/折叠元素称为“目标”，目标初始状态是折叠的，样式不作兼容处理。下面给出一些方案供参考：

  1. 将目标的 margin-top 设为 -100% 以使内容隐藏，
    展开时将 margin-top 设为 0。
    Demo

  2. 将目标的 max-height 设为 0 以使内容隐藏，
    展开时将 max-height 设为一个任意大于目标实际展开高度的值即可。
    Demo

  3. 将目标的 Y 轴缩放比例设为 0 ， 即 transform: scaleY(0); 这样内容就不可见了，
    展开时再将缩放比例设为 1 即可展开。
    Demo
    Demo2

以上三种方式都能实现动画效果。但是呢，在移动终端你会发现，方式1和2实现的动画效果就像是用砂纸打磨金属，并没有那么的顺畅。这是因为，相较于PC，移动设备的性能会差很多，方式1和2都会造成 Layout 的重绘。而方式3并未引起 Layout 的重绘，可以看到虽然目标元素收起时不可见，但是它却占据了空间以至于不会影响其他元素，所以动画效果要流畅许多。有关这部分的知识可参考这篇文章： High Performance Animations
